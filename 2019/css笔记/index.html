浮动

改变块元素的排列方式，让块元素水平排列

文档流（元素类型的排列方式）

块元素：每一个个内容都是占位的，上从往下垂直排列

内联元素（行内块元素）：每一个元素也都是占位，从左到右水平排列

脱离文档流（元素漂浮，不占位）

  • float：
  • left 左浮动 right 右浮动 none 不浮动

清除浮动

  • clear：
  •     left   清除左浮动
    right  清除右浮动
    both   清除两边浮动

    浮动的作用

    1、如果结构内容全部水平排列，全部内容都设置浮动 2、元素设置浮动后，脱离文档流不占位，会影响同级的下一个元素，可能会产生重叠效果 注意：盒子与盒子会重叠，与文本，图片，表单不重叠

浮动对元素类型的影响

    块元素： (原来块元素不设置width，块元素的宽度占满父元素的宽)
           1、浮动后, 块元素的宽度自适应受影响，宽度由内容撑开

    内联元素：
            1、设置浮动后可以加宽高
            2、浮动可以取消内联元素换行产生的间隙问题

    行内块
            1、浮动可以取消内联元素换行产生的间隙问题

盒模型的组成

一共四个部分
 内容、
 内边距（内填充padding）、
 边框线（border）、
 外边距margin

内边距 padding

1、内边距是设置盒子与内容之间的位置关系
2、设置内边距会撑大盒子，盒子会变形，想要恢复原来的大小，加上多少padding，在自身的宽高基础上减去多少。
3、padding的设置
        单一方向的padding
                    padding-left
                    padding-right
                    padding-top
                    padding-bottom

        多个方向的padding
                    1个值：4个方向
                    2个值：上下     左右
                    3个值： 上     左右      下
                    4个值： 上  右    下   左   4个方向

 4、padding不可以设置负值

外边距margin

1、外边距margin是设置盒子与盒子之间的位置关系

2、margin的设置

       单一方向的margin
                    margin-left
                    margin-right
                    margin-top
                    margin-bottom

        多个方向的padding
                    1个值：4个方向
                    2个值：上下     左右
                    3个值： 上     左右      下
                    4个值： 上  右  下   左   4个方向

3、块元素水平居中（跟浮动冲突）

margin:0 auto;
正常情况下，auto只能设置水平，自动居中，除非特殊情况

4、margin可以设置负值 水平方向 margin-left： 正值： 盒子往右跑 负值： 盒子往左跑

           margin-top:
                    正值：往下跑
                    负值：往上拉

5、margin常见的bug

1、清除浮动的元素设置margin-top的时候，需要先把上面的距离填满，才往下移动
        解决方法
                1、把margin-top换成给上面的浮动元素设置margin-bottom
                2、或者给浮动的元素套层父元素，用高度隔开（就不需要用clear设置）

2、上下两个盒子之间的margin会重叠（左右不会），谁的值大设置谁
        解决方法
                1、BFC解决（以后讲）
                2、只给一个盒子设置margin

3、margin-top的传递问题
父元素下的第一个子元素，设置margin-top的时候，margin-top会错误的传递给父元素，这个就是margin-top的传递问题
        解决方法
                父元素
                1、设置border-top
                2、设置padding-top
                3、overflow：hidden  设置内容超出隐藏

                4、  设置float   不推荐专门解决，可以顺便解决

                子元素
                    设置浮动float

                        content-box    标准盒子（默认值）

标准盒模型的计算
    盒子的总宽度：width+左右padding+左右的border
    盒子的总高度：height+上下padding+上下的border

    盒子的总宽度：width（包含padding和border）
    盒子的总高度：height（包含padding和border）

高度塌陷
    如果父元素的高度由内容撑开，内容浮动后不占位，父元素的高度就会塌陷

解决方案

        1、父元素设置overflow:hidden
                原理：overflow可以触发BFC（BFC的布局规则定义：浮动元素也参与高度计算）

        2、浮动元素最下方设置一个空盒子，给空盒子清除浮动，把塌陷的父元素撑开

BFC

BFC 格式化上下文，一个独立的渲染区域 ， 一个封闭的大盒子，有自己的布局规则

BFC的布局规则

1、BFC的盒子都是上下显示，垂直排列
2、同一个BFC的两个盒子，上下的margin会重叠
3、盒子左边的边框和margin会接触
4、BFC区域和浮动区域不会重叠
5、每一个BFC都是独立的，相互之间不影响
6、在BFC区域内，浮动元素也参与高度计算

BFC的触发条件

定位：绝对和固定
浮动：

            inlin-block

应用
    1、BFC解决高度塌陷
    2、BFC可以实现自适应两栏布局
    3、BFC可以解决margin重叠的问题
    4、BFC也可以解决margin传递问题

animation和transition的区别？

相同点：都是随着时间改变元素的属性值。
不同点：
transition需要触发一个事件(hover事件或click事件等)才会随时间改变其css属性；
而animation在不需要触发任何事件的情况下也可以显式的随着时间变化来改变元素css的属性值，从而达到一种动画的效果，css3的animation就需要明确的动画属性值。

响应式布局

设计特点：
面对不同分辨率设备灵活性强 
能够快捷解决多设备显示适应问题


缺点：
兼容各种设备工作量大，效率低下
代码累赘，会出现隐藏无用的元素，加载时间加长
其实这是一种折中性质的设计解决方案，多方面因素影响而达不到最佳效果
一定程度上改变了网站原有的布局结构，会出现用户混淆的情况


响应式布局核心
1、弹性图片
2、媒体查询（可以在不同设置的分辨率，改变内容的样式）

媒体查询

    /* 竖屏 */
      @media screen and (orientation: portrait) {
        body {
          background: green;
        }
      }

      /* 横屏 */
      @media screen and (orientation: landscape) {
        body {
          background: red;
        }
      }

移动端的设置

禁止窗口内容缩放，内容宽度等于当前设置宽

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
    width = device-width：宽度等于当前设备的宽度
    initial-scale： 初始的缩放比例（默认设置为1.0）
    minimum-scale：允许用户缩放到的最小比例（默认设置为1.0）
    maximum-scale：允许用户缩放到的最大比例（默认设置为1.0）
    user-scalable：用户是否可以手动缩放（默认设置为no，因为我们不希望用户放大缩小页面）

等比缩放布局

1、媒体查询结合rem

    @media all and (max-width: 320px) {
        html {
          font-size: 42px;
        }
      }

      @media all and (min-width: 321px) and (max-width: 376px) {
        html {
          font-size: 50px;
        }
      }
      @media all and (min-width: 377px) {
        html {
          font-size: 55px;
        }
      }

通过媒体查询的范围设置html的font-size，  内容的单位统一转换成 rem

    原理：  html的font-size值控制  单位 rem的内容


比如设计图是 750px     盒子测量 100px      dpr2
     1、 100px  /   dpr2   =50px

   （ 因为是375px   font-size  除以50px)
    2、 50px / 50px  =1rem   


2、vw+rem

    分两种情况：
            第一种直接转换  vw

                比如设计图  750px    逻辑375px     dpr2，

                    375px  =  100vw

                如果测量单位是100px    
                100px / 2   =50px
                50px /  375px  =13.33vw


          第二种找中间单位  rem （目的好算）
                比如设计图  750px    逻辑375px     dpr2

                1、确定html的font-size值
                    100px / 375px =  26.6666666667vw;
                    1rem = 100px

                如果测量单位是100px   
                    100px / 2   =50px
                    50px  / 100px = 0.5rem; 

单位：
em  ：先相对于自己，自己没有就相对于父元素font-size值

rem ：相对于根元素的font-size值  

vw :视口宽度的百分比
vh：视口高度的百分比
vmin：vh和vw谁小设置谁
vmax：vh和vw谁大设置谁
